Module 1: ISPS Code Purpose & Structure

What is the ISPS Code?

The International Ship and Port Facility Security (ISPS) Code is a comprehensive set of measures to enhance the security of ships and port facilities. It was adopted following the 9/11 attacks and became mandatory under the International Convention for the Safety of Life at Sea (SOLAS).

Purpose of ISPS Code:

  • Establish an international framework for maritime security
  • Prevent unlawful acts against ships and port facilities
  • Protect crew, cargo, and port workers
  • Ensure continuity of maritime commerce
  • Provide early detection and response to security threats

Structure of ISPS Code:

The ISPS Code consists of two parts:

  • Part A (Mandatory): Establishes security requirements and objectives
  • Part B (Recommended): Provides guidance on implementation

Key Principles:

  • Risk-based approach to security
  • Proportionate response to threats
  • Cooperation between ships and port facilities
  • Information sharing and communication
  • Continuous assessment and improvement

Module 2: Security Levels (MARSEC 1, 2, 3)

Understanding Maritime Security Levels:

The ISPS Code defines three security levels that reflect the degree of risk of a security incident. These levels determine the security measures that must be implemented on the ship.

MARSEC Level 1 (Green) - Normal Operations

Baseline security measures are in place
  • Normal port operations
  • Standard access control procedures
  • Routine security awareness
  • Regular security patrols
  • Identification checks at gangway

MARSEC Level 2 (Yellow) - Heightened Readiness

Enhanced security measures implemented
  • Increased surveillance and monitoring
  • Restricted access to designated areas
  • Enhanced crew briefings
  • Increased security patrols
  • Verification of all persons boarding
  • Search of baggage and cargo

MARSEC Level 3 (Red) - Maximum Security

Maximum security measures implemented
  • Continuous surveillance
  • Severe restrictions on access
  • Mandatory crew assembly
  • Full inspection of all persons and cargo
  • Possible relocation or rerouting
  • Coordination with authorities

Module 3: Ship Security Plan & SSO Responsibilities

Ship Security Plan (SSP):

Every ship must have a Ship Security Plan that outlines the security measures to be implemented on board. The SSP is confidential and must be kept secure.

Contents of a Ship Security Plan:

  • Ship and company identification
  • Ship Security Officer (SSO) details
  • Company Security Officer (CSO) details
  • Identification of restricted areas
  • Access control procedures
  • Cargo and baggage handling procedures
  • Crew identification and verification
  • Visitor management procedures
  • Delivery of supplies and provisions
  • Security incident reporting procedures
  • Security drill and training procedures
  • Cyber security measures

Ship Security Officer (SSO) Role:

The SSO is responsible for implementing and maintaining the Ship Security Plan on board. Key responsibilities include:

  • Conducting security assessments
  • Implementing security measures
  • Training and awareness programs
  • Coordinating with port facilities
  • Reporting security incidents
  • Maintaining security records
  • Conducting security drills
  • Liaison with authorities

SSO Qualifications:

  • Appropriate maritime training and experience
  • STCW A-VI/5 certification
  • Knowledge of ISPS Code
  • Understanding of threat assessment
  • Communication and leadership skills

Module 4: Access Control, Drills & Incident Reporting

Access Control Procedures:

Effective access control is fundamental to ship security. All persons boarding the ship must be identified and verified.

Gangway Management:

  • Gangway watch 24/7 when in port
  • Identification of all persons boarding
  • Verification against crew lists
  • Visitor passes and escort procedures
  • Removal of passes upon departure
  • Recording of all persons on board

Restricted Areas:

  • Engine room
  • Bridge
  • Cargo holds
  • Communication equipment areas
  • Clearly marked with warning signs
  • Access restricted to authorized personnel

Security Drills:

Regular security drills are mandatory to ensure crew readiness and familiarity with procedures.

  • Frequency: At least once every 3 months
  • Content: Simulated security incidents
  • Participation: All crew members
  • Documentation: Drill records and observations
  • Improvements: Lessons learned and corrective actions

Security Incident Reporting:

  • Any suspicious activity must be reported immediately
  • Report to SSO or Master
  • Notify port state authorities if required
  • Document all details of the incident
  • Preserve evidence
  • Cooperate with investigations

Module 5: Port Facilities & Maritime Cyber Security

Interaction with Port Facilities:

Ships must coordinate security measures with port facilities to ensure comprehensive security coverage.

Declaration of Security (DoS):

  • Document exchanged between ship and port facility
  • Confirms security measures are in place
  • Required when ship is at security level 2 or 3
  • Signed by Master and Port Facility Security Officer
  • Kept on file for audit purposes

Port State Control (PSC):

  • Inspections by port state authorities
  • Verification of ISPS Code compliance
  • Review of Ship Security Plan
  • Inspection of security measures
  • Deficiencies must be corrected

Maritime Cyber Security:

Cyber security is now an integral part of the Ship Security Plan. Modern ships rely on digital systems for navigation, communication, and cargo management.

Cyber Security Threats:

  • Unauthorized access to ship systems
  • Malware and ransomware attacks
  • Data theft and espionage
  • Disruption of critical systems
  • Communication interception

Cyber Security Measures:

  • Firewalls and intrusion detection systems
  • Strong password policies
  • Regular software updates and patches
  • Network segmentation
  • Crew training on cyber awareness
  • Incident response procedures
  • Backup and recovery systems

Officer Responsibilities:

  • Maintain awareness of cyber threats
  • Report suspicious activity immediately
  • Follow password and access control procedures
  • Participate in cyber security training
  • Protect sensitive information
